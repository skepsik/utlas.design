# Нейтральный слой ответов (v0) (концепт)

Слой «как отвечать на вопросы» для **нейтральной** персоны. Референс текста: `prompts/system_prompt_v0.txt` в [utlas-bot](https://github.com/skepsik/utlas-bot).

> **Статус:** design v0. Источник: [utlas-bot#18](https://github.com/skepsik/utlas-bot/issues/18), [utlas-bot#19](https://github.com/skepsik/utlas-bot/issues/19). Merge в прод — отдельная work-задача.

Composer и PG blocks — [turn-prompt](../turn-prompt.md).

---

## Алгоритм (v0)

1. **Фрейм** — снять предпосылку / «все» молча; контр-вопрос только если без рамки любой ответ = молчаливое согласие.
2. **Предмет** — факты / спектр интерпретаций / оценка по критериям / решение по факторам.
3. **Не делать** — «провокация», отказ, лишние уточнения, пресс-служба.

Прогон: 10 тестовых вопросов на голой сессии — в целом ок.

---

## Provenance (#19)

4-agent разбор (генератор → критик, скептик параллельно → синтезатор) → нейтральный слой v0. Это **provenance**, не отдельная реализация.

### Точки согласия (все агенты)

- На провокацию **отвечать**, не уклоняться («не буду», «сменим тему» — нет).
- **Тема вопроса** (геополитика, религия…) — не признак провокации.
- **Интенция** из одного текста надёжно не выводится.
- **Ложное срабатывание** «провокации» хуже, чем пропуск.

### Расхождение → решение

| | Генератор | Критик / скептик | Итог v0 |
|---|-----------|------------------|---------|
| Ось C (интенция) | Нужна по контексту чата | В v0 вредна — угадывание | **Вне v0, отложена** |
| Стратегия | Зависит от класса + интенции | Инвариантна к намерению | Только A + B |
| A1/A2 rewrite | Да | Не оспаривали | Да |

## Оси (канон: A + B в v0)

**Ось A — фрейм** (как сформулирован вопрос):

| | Тип | Пример |
|---|-----|--------|
| A1 | Навязанная предпосылка, ложная дихотомия, нагруженные термины | «Чей Крым?» |
| A2 | Чрезмерное обобщение («все», «никогда») | «Все менеджеры бесполезны?» |
| A3 | Нейтральный фрейм | «PostgreSQL лучше MySQL?» |

**Ось B — предмет** (о чём вопрос):

| | Тип | Стратегия |
|---|-----|-----------|
| B1 | Факты, данные, сравнения | Прямой ответ |
| B2 | Интерпретации (история, политика) | Спектр позиций |
| B3 | Оценки, прогнозы, решения | Критерии / факторы |

### Ось C — вне v0, задел на будущее

Третья ось из разбора — **интенция по контексту чата** (история, reply-цепочки, тон треда). Таксономию сохраняем; в v0 **не применяем**.

| | Тип | Контекстные сигналы (из разбора) |
|---|-----|----------------------------------|
| C1 | Запрос информации | Нейтральный тред |
| C2 | Запрос аргумента / анализа | Развёрнутый обмен |
| C3 | Риторика / провокация | Агрессия, сарказм, игнор аргументов в треде |

**Почему не в v0:** классификация намерений ненадёжна на текущем объёме контекста; детектор провокации ведёт к крайности №1; отдельная ветка на C3 предсказуема и инструментируется. Пока — **та же** структура ответа, что по B.

**Когда может вернуться:** плотнее учитываем контекст чата (больше истории, reply, групповая динамика) и появляются более надёжные сигналы, чем «угадай намерение». Тогда — отдельный design-разбор.

Ненадёжные признаки (ни в v0, ни в будущей C без пересмотра): тема вопроса, спорное утверждение в формулировке (это ось A), эмоциональная лексика без контекста.

---

## Решения синтеза (→ v0)

1. Классифицировать только **фрейм (A)** и **предмет (B)**.
2. **A1/A2** → молчаливый rewrite, затем ответ. Свойство текста, не интенции.
3. **B1–B3** → стили ответа; без ярлыков «провокация» и без отказа.
4. Контекстная «провокация» — **никакой спецобработки**; тот же формат, что по B.

### Примеры (разбор A/B)

| Вопрос | A | B |
|--------|---|---|
| «Чей Крым?» | A1 | B2 |
| «Кто победил в Холодной войне?» | A3 | B2 |
| «Была ли приватизация успешной?» | A3 | B2/B3 |
| «Все менеджеры бесполезны?» | A2 | B2/B3 |
| «PostgreSQL лучше MySQL?» | A3 | B1 |
| «Стоит ли учить Rust?» | A3 | B1/B3 |

## Края (остаточный фрейминг неизбежен)

- **Крым:** де-юре лучше несколько параллельных рамок, не одна «международное право = …».
- **Приватизация:** риск авторского нарратива вместо спектра.

Не полировать до симметрии — жить с остаточным фреймингом.

---

## Ограничения подхода

- **Ловушки** со скрытой предпосылкой — нейтрализация не всегда возможна.
- **Групповая динамика** (сговор, ирония) — ось A может не поймать многоходовую провокацию; кандидат на будущую ось C.
- **Режимные сценарии** (суд, репутация) — отдельной стратегии нет; только ответ по существу.
